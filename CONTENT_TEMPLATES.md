### 実績紹介のフォーマット

```markdown
## [プロジェクト名]

### 課題
- [具体的な課題1]
  例：ページ読込時間が3.5秒と遅く、モバイルでの離脱率が高い
- [具体的な課題2]
  例：商品画像の最適化が不十分
- [具体的な課題3]
  例：Core Web Vitalsで赤評価

### 実施した施策
- [実施内容1]
  例：画像の次世代フォーマット（WebP）への変換と遅延読込実装
- [実施内容2]
  例：Code Splittingによるバンドルサイズ削減
- [実施内容3]
  例：不要なサードパーティスクリプトの削除

### 成果
- **[数値]%** [指標名]
  例：66%読込時間短縮（3.5秒→1.2秒）
- **[数値]%** [指標名]
  例：25%離脱率改善
- **[数値]%** [指標名]
  例：18%CV率向上

### 工数・期間
約[XX]時間 / [Y]週間

### 使用技術
React, TypeScript, Next.js, Webpack
```

### 実績の書き方のポイント

```
✅ 良い例：
「読込時間を3.5秒から1.2秒に短縮（66%改善）。
離脱率25%改善、CV率18%向上を実現。」

❌ 悪い例：
「パフォーマンスを改善しました。
お客様に喜んでいただけました。」

【ポイント】
1. 具体的な数字を入れる
2. Before/Afterを明示
3. ビジネス成果（CV率、売上等）を含める
4. 実施期間・工数も記載
```
