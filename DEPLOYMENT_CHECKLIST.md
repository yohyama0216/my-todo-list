# 公開前チェックリスト

このチェックリストを使用して、サイトを公開する前に重要な項目を確認してください。

## ✅ 必須項目（公開前に必ず確認）

### コンテンツ

- [ ] **3秒ルール**: トップページを開いて3秒以内に「何ができるか」が分かる
- [ ] **実績に数字**: 少なくとも3つの実績に具体的な数字（改善率、期間等）がある
- [ ] **料金目安**: 明確な料金目安が記載されている
- [ ] **CTAボタン**: すべてのページに問い合わせ導線（CTAボタン）がある
- [ ] **問い合わせフォーム**: フォームが正常に動作する（テスト送信済み）

### 技術・SEO

- [ ] **ページタイトル**: すべてのページに適切な `<title>` タグが設定されている
- [ ] **メタディスクリプション**: すべてのページに `description` が設定されている（160文字以内）
- [ ] **OGP設定**: OGP画像とメタタグが設定されている
- [ ] **robots.txt**: robots.txt が存在し、適切に設定されている
- [ ] **sitemap.xml**: サイトマップが生成されている
- [ ] **favicon**: ファビコンが設定されている
- [ ] **404ページ**: 存在しないページへのアクセス時の動作を確認

### デザイン・UX

- [ ] **レスポンシブ**: スマートフォン、タブレット、PCで正しく表示される
- [ ] **読みやすさ**: フォントサイズ、行間、余白が適切
- [ ] **リンク動作**: すべてのリンクが正しく動作する
- [ ] **画像表示**: すべての画像が正しく表示される
- [ ] **画像alt属性**: すべての画像に alt テキストが設定されている

### パフォーマンス

- [ ] **読込速度**: ページが3秒以内に表示される
- [ ] **Lighthouseスコア**: パフォーマンススコアが80以上
- [ ] **画像最適化**: 画像が適切なサイズ・フォーマットで配信されている

## 🎯 推奨項目（可能な限り対応）

### コンテンツの質

- [ ] 自己紹介が具体的（「頑張ります」等の抽象表現を避けている）
- [ ] サービス説明に具体例がある
- [ ] FAQが5つ以上ある
- [ ] 各ページのテキストが読みやすい（箇条書き、見出しを活用）

### SEO・アクセシビリティ

- [ ] 構造化データ（JSON-LD）が設定されている
- [ ] 見出しタグ（h1, h2, h3）が正しく使われている
- [ ] コントラスト比が適切（WCAG AA基準）
- [ ] キーボードだけで操作できる
- [ ] フォーカス表示が分かりやすい

### アナリティクス

- [ ] Google Analytics（または代替ツール）が設定されている
- [ ] コンバージョン計測が設定されている
- [ ] Search Consoleに登録されている

## 🚀 公開手順

### 1. 最終確認

```bash
# ローカルでビルドして確認
npm run build
npm run preview

# ブラウザで http://localhost:4321 を開いて確認
```

### 2. Formspreeの設定

1. [Formspree](https://formspree.io/)にログイン
2. 新しいフォームを作成
3. フォームIDをコピー
4. `src/pages/contact.astro` の以下の部分を更新：

```html
<form 
  action="https://formspree.io/f/あなたのフォームID" 
  method="POST"
>
```

5. 変更をコミット・プッシュ

### 3. GitHub Pagesの設定確認

1. GitHubリポジトリの Settings → Pages を開く
2. Source が "GitHub Actions" になっているか確認
3. 設定されていない場合は、"GitHub Actions" を選択

### 4. デプロイ

```bash
# 変更をコミット
git add .
git commit -m "サイト公開準備完了"

# メインブランチにプッシュ（またはプルリクエストをマージ）
git push origin main
```

### 5. デプロイ完了確認

1. GitHubの Actions タブでワークフローが成功したか確認
2. 公開URLにアクセスして動作確認
   - `https://あなたのユーザー名.github.io/リポジトリ名/`

### 6. テスト送信

- お問い合わせフォームからテスト送信
- メールが正しく届くか確認

## 🔍 公開後チェック

### 初日

- [ ] すべてのページが表示される
- [ ] 画像が正しく表示される
- [ ] リンクが正しく動作する
- [ ] フォーム送信が機能する
- [ ] モバイルで正しく表示される

### 1週間以内

- [ ] Search Consoleでインデックス状況を確認
- [ ] OGP画像がSNSで正しく表示されるか確認（TwitterやFacebookでシェアテスト）
- [ ] Google Analytics でアクセスが記録されているか確認
- [ ] 問い合わせフォーム経由でメールが届くか実際にテスト

### 1ヶ月以内

- [ ] Search Consoleで検索パフォーマンスを確認
- [ ] Core Web Vitalsのスコアを確認
- [ ] 実際の問い合わせ数を確認
- [ ] 必要に応じてコンテンツを更新

## 🛠️ トラブルシューティング

### ページが表示されない

1. GitHub Actions のワークフローログを確認
2. ビルドエラーがないか確認
3. `astro.config.mjs` の `site` と `base` 設定を確認

### 画像が表示されない

1. 画像パスが正しいか確認（`public/` からの相対パス）
2. 画像ファイルが実際に存在するか確認
3. ファイル名の大文字・小文字を確認

### フォームが送信できない

1. Formspree のフォームIDが正しいか確認
2. ブラウザの開発者ツールでエラーを確認
3. Formspreeのダッシュボードで送信履歴を確認

### スタイルが崩れる

1. ビルドを再実行（`npm run build`）
2. ブラウザのキャッシュをクリア
3. 異なるブラウザで確認

## 📊 定期メンテナンス

### 毎月

- [ ] アクセス解析を確認
- [ ] 問い合わせ数を確認
- [ ] リンク切れがないかチェック
- [ ] 実績を更新（新しいプロジェクトがあれば）

### 3ヶ月ごと

- [ ] 依存パッケージの更新（`npm update`）
- [ ] セキュリティアップデート確認（`npm audit`）
- [ ] コンテンツの鮮度を確認（古い情報がないか）

### 6ヶ月ごと

- [ ] デザインのトレンドチェック
- [ ] 競合サイトの調査
- [ ] ユーザーフィードバックの反映

## 🎯 成果測定の指標

### 定量指標

- ページビュー数
- 問い合わせ数
- 問い合わせからの成約率
- 平均セッション時間
- 直帰率

### 定性指標

- 問い合わせの質（具体的なプロジェクトの相談か）
- 初回問い合わせでの情報の充実度
- 問い合わせ者の予算感

## 💡 改善のヒント

### 問い合わせが少ない場合

1. CTAボタンが目立つか確認
2. 料金が明確か確認
3. 実績に具体的な数字があるか確認
4. お問い合わせのハードルを下げる（「まずは相談だけでも」等）

### 直帰率が高い場合

1. 3秒以内に「何屋か」分かるか確認
2. ファーストビューの改善
3. 読込速度の改善
4. コンテンツの見直し

### 問い合わせはあるが成約しない場合

1. 期待値と実力のギャップがないか確認
2. 料金設定の見直し
3. 実績の質を高める
4. 問い合わせ時の対応を改善

---

**公開は完璧である必要はありません。**
最低限のチェック項目をクリアしたら公開し、実際のフィードバックを得ながら改善していくことが重要です！
